Move files and APKs between Windows and Quest.

Browse the storage that Android exposes through ADB, copy files in either direction, install your own APKs on one or several headsets, retrieve sideloaded single-APK apps, and optionally manage Rusty Kiosk.

Install the app

  1. Download the guided setup

    Save MetaQuestFileManager-Setup.exe from the button above, then open it.

  2. Review the Windows prompts

    Approve the administrator prompt only if the publisher and app name match this project. The helper explains that it will trust the public package certificate and install the update feed.

  3. Finish setup

    The app installs under the stable Meta Quest File Manager identity and launches when setup completes. Later releases can update through the registered App Installer feed.

  4. If setup is blocked

    Download the certificate and install it for the local machine in Trusted People, then open the App Installer file. If package installation is restricted, extract the portable ZIP and run MetaQuestFileManager.exe.

Prepare Windows and your Quest

  1. Install Android SDK Platform Tools

    The app uses adb.exe but does not redistribute it. Get the current Windows package from Android Developers. If you do not use Android Studio, extract its platform-tools folder under %LOCALAPPDATA%\Android\Sdk; the app checks that location automatically.

  2. Enable Developer Mode

    Follow Meta's device setup guide for your headset and account. This app cannot enable account-level Developer Mode for you.

  3. Connect by USB

    Use a USB cable that supports data. Put on the headset and approve its USB debugging prompt. Choose “always allow” only on a PC you trust.

  4. Check the connection

    Open Meta Quest File Manager and select Refresh. A ready headset appears in the selector at the top. If it says unauthorized, return to the headset and approve the debugging prompt.

Use the app

Select the headset first. Every file and APK action is then scoped to that device unless you explicitly choose several Wi-Fi headsets.

Browse and copy files

Open the Files tab. Browsing starts at /sdcard, the shared-storage tree that includes folders such as Download, DCIM, and Movies. Double-click folders, pull a selected file to Windows, or push a Windows file into the open folder.

Install one APK

Open APKs, browse to an APK, review the install options, and choose Install on selected headset. The status bar shows honest indeterminate progress while ADB works.

Install an APK bundle

Choose a folder containing the base APK and every required split APK. The app sends all top-level .apk files together as one Android package set; it does not install unrelated apps one by one.

Export a sideloaded APK

Refresh the third-party package list, select a package, and choose Export selected package. Export succeeds only when Android reports one APK file and also writes a SHA-256 checksum.

Check power and performance

Open Device to read headset and controller battery state. Keep-awake and fixed CPU/GPU controls are explicit ADB maintenance actions with a restore or clear route.

Use Rusty Kiosk only if wanted

The Rusty Kiosk tab is optional. Files, APKs, Wi-Fi ADB, and device status continue to work when Kiosk is absent or never installed.

Connect and install over Wi-Fi

  1. Start with USB

    Keep the PC and headset on the same reachable network. Connect and authorize the headset by USB, then select it at the top of the app.

  2. Enable Wi-Fi ADB

    Open Wi-Fi ADB, leave port 5555 unless your setup requires another port, and choose Enable and connect. Review the confirmation before the debugging transport changes.

  3. Refresh and select

    The wireless connection appears with a [Wi-Fi] label. Repeat the USB enablement for each headset you want to use, or connect an already-enabled headset by its local IPv4 address.

  4. Install on several headsets

    In APKs, check at least two Wi-Fi headsets, choose one APK or one bundle folder, set a sensible parallel limit, and use the checked-headsets install button. Progress and results remain separate for each target.

Wi-Fi ADB may stop after a headset reboot, network change, or authorization change. Reconnect by USB and enable it again when needed. The app does not restart the global ADB server.

Optional Rusty Kiosk setup

Rusty Kiosk is a separate open-source Spatial SDK app. It adds an on-headset app catalogue, tags, normal and guarded launch, passthrough controls, and an optional direct PC link. Nothing in this section is required for ordinary file-manager use.

  1. Connect once by USB

    Authorize the headset as described above, select it in the app, open Rusty Kiosk, and choose Install and provision (USB). The app installs the main APK and its same-signer setup helper, grants only the helper's narrow setup permission, and confirms the resulting package state.

  2. Choose the capabilities you want

    Inside Rusty Kiosk, open User controls. Accessibility enables the soft guard; the direct PC link enables local Kiosk management without routine ADB; Wi-Fi ADB remains a separate maintenance option. Each is visibly labeled and independently reversible.

  3. Pair the direct link

    On a trusted shared network or private Windows hotspot, enable Direct PC link. Enter its displayed address and pairing code in the Windows Kiosk tab, then connect. The code can be rotated from the headset at any time.

  4. Manage apps and tags

    Search the complete Kiosk catalogue, filter or edit tags, retain named entries for apps that are not installed, import or export the tag file, and choose normal or guarded launch. The same fixed operations are available through USB/Wi-Fi ADB or the paired direct link.

  5. Prefer ADB for batch APK installs

    The normal APK tab can install several packages without repeated in-headset approval after the PC is authorized. Kiosk's direct installer is an attended fallback: Android may still ask the wearer to confirm each app installation session.

Soft guard, not lock task

The first two Home presses restore a guarded app. A third within five seconds returns to Rusty Kiosk. From Rusty Kiosk, Home reaches Meta Home normally.

Confirmed state

PC mutations progress through sent, pending, and confirmed. A prompt or successful command alone is not treated as effective headset state.

Read the dedicated Rusty Kiosk onboarding guide or inspect the AGPL-licensed source.

Common questions

No headset appears after Refresh

Confirm Developer Mode, use a data-capable cable, try another USB port, and check inside the headset for the debugging approval prompt. Also confirm that the footer does not say “ADB not found.”

The footer says ADB not found

Install Android SDK Platform Tools in the standard Android SDK location, add its folder to PATH, or set META_QUEST_FILE_MANAGER_ADB to the full path of adb.exe, then reopen the app.

An APK installs but does not run

The APK may target a different CPU or Android version, require missing assets or services, or depend on another package. Copying an APK does not copy saves, app data, OBB files, downloaded assets, purchases, or store entitlements.

Why is split-package export refused?

Modern apps can consist of a base APK plus several split APKs. The current export route intentionally refuses those packages so it never presents an incomplete backup as usable.

Can it browse the whole Quest filesystem?

No. Quest has a normal Android folder hierarchy, but ADB shell access is permission-limited. The app shows only paths Android permits and begins in shared storage at /sdcard.

Do I need Rusty Kiosk or the direct link?

No. They are optional. The normal file browser, APK tools, Wi-Fi ADB, and device controls remain available through an authorized ADB connection without installing or enabling Rusty Kiosk.

Why is a Kiosk action still pending?

Some actions require a visible Meta or Android decision. The PC keeps them pending until the headset reports the requested effective state, and a later refresh can reconcile the result.
